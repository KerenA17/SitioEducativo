¡Conceptos Básicos!


¿Qué es una base de datos?

Una base de datos se refiere al conjunto de datos que se almacenan de manera electrónica. El propósito principal es mantener la información de manera ordenada para permitir que se realicen acciones como las consultas, modificaciones, eliminación de los datos almacenados.

Sistema de gestión de bases de datos

El software es el medio por el cual se crea, lee, actualiza y elimina los datos que conforman la base de datos.



Elementos que conforman una base de datos
Clave primaria (llave primaria).

Es un campo o conjunto de campos que tienen valores únicos; son datos que sirven para identificar de manera única cada registro.

Clave foránea (llave foránea).

Es un campo de una tabla que apunta a la llave primaria de otra tabla. Su función principal es crear un vínculo entre las tablas para relacionar los datos distribuidos.

Relación.

Es la conexión o enlace que se establece entre dos o más tablas a través de claves primarias y foráneas. Estas relaciones permiten organizar la información de forma que se pueda consultar y gestionar eficientemente.

Entidad.

Es cualquier objeto, persona, lugar o concepto sobre el cual queremos almacenar información en la base de datos. Cada entidad se representa en la base de datos como una tabla, y cada tabla contiene los datos relacionados a esa entidad.

Atributo.

Son las propiedades o características que describen a una entidad. Cada atributo se convierte en una columna dentro de la tabla que representa la entidad.

Estructura de una base de datos relacional
Tabla.

Las bases de datos se construyen a partir de tablas. Cada tabla representa una entidad principal sobre la cual se registran datos específicos.

Columna.

Las columnas, también llamadas atributos, definen las características más importantes de la entidad. Cada columna almacena un tipo de dato particular, como nombre, edad o dirección.

Registro.

Un registro es una fila en la tabla que contiene los valores específicos de los atributos para una entidad. Cada fila representa un elemento único dentro de la base de datos.

Tipos de relaciones
1. Relación uno a uno (1:1)

Un registro de una tabla está relacionado con UN SOLO registro de otra tabla, y viceversa.
Ejemplo: Cada persona tiene UN pasaporte, y cada pasaporte pertenece a UNA persona.

2. Relación uno a muchos (1:N)

Un registro de una tabla puede estar relacionado con VARIOS registros de otra tabla, pero cada registro de la segunda tabla solo pertenece a UNO de la primera.
Ejemplo: Un cliente puede hacer MUCHOS pedidos, pero cada pedido pertenece a UN solo cliente.

3. Relación muchos a muchos (N:N)

Un registro de una tabla puede estar relacionado con varios registros de otra tabla, y viceversa.
Ejemplo: Un estudiante puede estar inscrito en VARIOS cursos, y cada curso puede tener MUCHOS estudiantes.
